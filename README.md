# C++ Demo App & Snyk Open Source CI/CD Actions 🛡️

This repository is a demonstration environment for scanning C/C++ projects using **Snyk**. It features automated CI/CD workflows for unmanaged scanning and analyzing Software Bill of Materials (SBOM) generated by Snyk.

---

## ⚠️ Important Notes

* **Dependency Scanning:** C/C++ scanning with Snyk requires the source code of the packages being leveraged. Typically, these must reside within the `/vendors` directory.
* **Authentication:** When you fork this repo, you **must** set up your repository environment variables. 
    * Add `SNYK_TOKEN` to your GitHub Actions Secrets to allow the workflows to authenticate with your Snyk account.

---

## 🚀 CI/CD Examples

Automated security gates using GitHub Actions leveraging both `snyk test --unmanaged` and ` snyk sbom test` using a Snyk generated SBOM

---

## 🛠️ Setup Instructions

### 1. Prerequisites
Ensure you have a Snyk account. If you don't, you can [sign up for free](https://snyk.io).

### 2. Configure GitHub Secrets
To enable the CI/CD workflows, follow these steps:
1. Go to your forked repository on GitHub.
2. Navigate to **Settings** > **Secrets and variables** > **Actions**.
3. Click **New repository secret**.
4. Name: `SNYK_TOKEN`
5. Value: *[Your Snyk API Token]*
